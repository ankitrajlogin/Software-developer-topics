## Adding Content / Contributing
- Fork the repo and create a branch: `git checkout -b feature/<topic>`
- Add or update files under the appropriate folder (docs/, notes/, examples/).
- Keep entries concise and focused; prefer examples and actionable commands.
- Submit a PR with a clear title and description.
- Follow existing naming and formatting conventions.

## Style and Naming Guidelines
- Use clear filenames and subfolders (lowercase, hyphen-separated).
- Prefer Markdown (.md) for notes and docs.
- Include short summaries and examples for each topic.

For questions or suggestions, open an issue or submit a pull request.
